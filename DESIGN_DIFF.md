# yadocho カレンダー画面 デザイン差異レポート

作成日: 2026-03-20
比較対象: モックアップ画像 vs デプロイ済み画面 (https://yadocho.com/calendar)
環境: iPhone SE 375×667 想定

---

## 1. ヘッダー

### 1-1.「今日」ボタン — 重要度: 高

| 項目 | モックアップ | 現在の実装 |
|------|------------|-----------|
| 背景色 | 赤/コーラル系 (`bg-accent` 相当) | 白 (`bg-surface`) |
| テキスト色 | 白 | グレー (`text-text-3` = `rgb(168,162,158)`) |
| 枠線 | なし（塗りつぶし） | あり (`border border-border`) |
| 形状 | ピル型角丸 | ピル型角丸 ✅ 一致 |

**修正案:**
```
- text-text-3 bg-surface border border-border
+ text-white bg-accent border-0
```
→ `bg-accent`（コーラル系）に塗りつぶし、テキスト白、ボーダー削除

### 1-2.「2024年7月 ▼」年月表示 — 重要度: 低

| 項目 | モックアップ | 現在の実装 |
|------|------------|-----------|
| フォントサイズ | やや大きめ (text-lg〜text-xl相当) | `text-base` (16px) |
| フォントウェイト | Bold | `font-bold` ✅ 一致 |
| ドロップダウン矢印 | ▼ 三角形 | ChevronDown SVG (16px) ✅ ほぼ一致 |
| 配置 | 中央 | 中央 ✅ 一致 |

**修正案:**
```
- text-base font-bold
+ text-lg font-bold
```
→ モックアップの方が若干大きく見える。`text-lg` (18px) に変更を検討

### 1-3.「⚙ 休設定」ボタン — 重要度: 中

| 項目 | モックアップ | 現在の実装 |
|------|------------|-----------|
| アイコン | ⚙ 歯車 (settings) | 🌴 ヤシの木 (`lucide-tree-palm`) |
| テキスト | 「休設定」 | 「休設定」 ✅ 一致 |
| スタイル | 枠線あり、角丸ピル | 枠線あり、角丸ピル ✅ 一致 |
| 背景 | 白/透明 | `bg-surface` (白) ✅ 一致 |

**修正案:**
```tsx
// アイコンを Settings (歯車) に変更
- <TreePalm size={14} />
+ <Settings size={14} />
```
→ `lucide-react` の `Settings` アイコンに差し替え

### 1-4. ヘッダー3要素の配置バランス — 重要度: 低

- モックアップ: 左（今日）・中央（年月）・右（休設定）で均等配置
- 現在: `flex items-center justify-between px-4 py-2.5` ✅ 概ね一致
- 微差: モックアップの方が左右パディングがやや広い印象

**修正案:** 必要に応じて `px-4` → `px-5` を検討

## 2. カレンダーグリッド

### 2-1. 日付ヘッダー — 重要度: 低

| 項目 | モックアップ | 現在の実装 |
|------|------------|-----------|
| 表示形式 | 数字 + 改行 + (曜日) | 数字 + 改行 + (曜日) ✅ 一致 |
| 今日の丸背景 | 赤/コーラル丸 | 赤/コーラル丸 ✅ 一致 |
| 土曜色 | 青系テキスト | 赤/コーラル系 ❌ |
| 日曜色 | 赤系テキスト | 赤/コーラル系 ✅ 一致 |

**修正案:**
```
// 土曜日の色を青系に変更
土曜日: text-accent → text-blue-500 or text-blue-600
```
→ モックアップでは土曜=青、日曜=赤が日本の慣習通り

### 2-2. 部屋名カラム — 重要度: 高

| 項目 | モックアップ | 現在の実装 |
|------|------------|-----------|
| 表示形式 | タイプ名+番号（例: シングル101, ダブル201） | 番号のみ（例: 201, 202） |
| フォントサイズ | やや大きめ | `text-xs font-bold` |
| 背景色 | 薄いグレー/ベージュ | 背景なし or 非常に薄い |
| 幅 | 広め（タイプ名が入る分） | 狭い |

**修正案:**
```
// 部屋名の表示を「タイプ名 + 番号」に変更（データ側の対応も必要）
"201" → "ダブル201"  等

// カラム幅を広げる
room column: min-w-[80px] → min-w-[100px] 程度

// 背景色を追加
+ bg-stone-50 or bg-surface
```
→ これはデータ構造の変更も伴うため、部屋マスタに「表示名」フィールドがあるか要確認

### 2-3. グリッドセルの「+ 予約作成」テキスト — 重要度: 高

| 項目 | モックアップ | 現在の実装 |
|------|------------|-----------|
| 表示テキスト | 「+ 予約作成」（フルテキスト） | 「＋」のみ |
| フォントサイズ | 小さめだが読める | 「＋」のみで非常にミニマル |
| 色 | グレー系 | グレー系 ✅ |

**修正案:**
```tsx
// セル内のテキストを変更
- <span>＋</span>
+ <span>＋ 予約作成</span>

// ただしセル幅が狭い場合はレスポンシブで出し分け
// モバイルでは「＋」のみ、タブレット以上で「＋ 予約作成」も検討
```
→ モックアップでは7日表示で「+ 予約作成」が見えているので、セル幅との兼ね合いを確認

### 2-4. 予約バーのスタイル — 重要度: 中

| 項目 | モックアップ | 現在の実装 |
|------|------------|-----------|
| 色 | 緑系・オレンジ系など複数色 | 赤/サーモン系の単色 |
| テキスト | 「名前 (人数)」＋ ✅アイコン | 「名前(人数)」のみ |
| 角丸 | やや角丸 (rounded-md相当) | 角丸あり ✅ |
| 複数日またぎ | バーが複数セルにまたがる | またがる ✅ |

**修正案:**
```
// 予約ステータスや部屋タイプ別に色分けを導入
confirmed: bg-green-100 text-green-800 (モックの緑バー)
pending: bg-amber-100 text-amber-800
checked-in: bg-accent-soft text-accent (現在のサーモン系)

// チェックマークアイコンを追加（confirmed予約に）
+ <CheckCircle size={12} className="shrink-0" />

// 現在のクラス
"absolute flex items-center gap-0.5 px-1.5 text-[11px] font-semibold truncate"
// に色分けロジックを追加
```

### 2-5. 休業日のハッチング柄 — 重要度: 中

| 項目 | モックアップ | 現在の実装 |
|------|------------|-----------|
| 表現 | 斜線ハッチング（グレー対角線パターン） | 確認できず（未実装 or データなし） |

**修正案:**
```css
/* 休業日セルに斜線パターンを適用 */
.cell-closed {
  background-image: repeating-linear-gradient(
    -45deg,
    transparent,
    transparent 4px,
    rgba(0,0,0,0.06) 4px,
    rgba(0,0,0,0.06) 5px
  );
}
```
→ Tailwind では `bg-[repeating-linear-gradient(...)]` or カスタムクラスで対応

### 2-6. グリッドの罫線・背景 — 重要度: 低

| 項目 | モックアップ | 現在の実装 |
|------|------------|-----------|
| セル罫線 | 薄いグレー実線 | `border-border/15` (非常に薄い) |
| 全体背景 | 白〜オフホワイト | 白系 ✅ ほぼ一致 |
| 罫線の視認性 | はっきり見える | かなり薄い |

**修正案:**
```
// 罫線の透明度を上げる
- border-border/15
+ border-border/30 or border-border/40
```

## 3. DayPanel（日付パネル）

### 3-1. 日付表示フォーマット — 重要度: 中

| 項目 | モックアップ | 現在の実装 |
|------|------------|-----------|
| フォーマット | 「2024年7月3日(水)」 | 「3/20（金）」 |
| ナビ矢印 | ◀ ▶ テキスト矢印 | < > ボタン |
| 「今日」バッジ | なし | あり（日付横にピルバッジ） |

**修正案:**
```tsx
// 日付フォーマットを変更
- "3/20（金）"
+ "2026年3月20日（金）"

// ナビ矢印のスタイル
// モックアップでは ◀ ▶ を使用しているが、現在の < > でも問題ない
// 統一するなら ChevronLeft / ChevronRight アイコンを使用
```

### 3-2. チェックイン/チェックアウトのレイアウト — 重要度: 高

| 項目 | モックアップ | 現在の実装 |
|------|------------|-----------|
| セクション表示 | 「チェックイン (3件)」括弧表記 | 「チェックイン — 5件」ダッシュ表記 |
| レイアウト | 左右2カラム（CI/CO並列） | 1カラム縦並び |
| リスト内容 | 時刻 + 名前（例: 15:00 田中太郎） | 名前様 + 部屋番号バッジ + 詳細 |
| アイコン | なし | LogIn / LogOut SVGアイコンあり |

**修正案:**
```
// セクションヘッダーのフォーマット
- "チェックイン — 5件"
+ "チェックイン（5件）"

// レイアウトを2カラムに変更
<div className="grid grid-cols-2 gap-4">
  <div>チェックイン（N件）...</div>
  <div>チェックアウト（N件）...</div>
</div>

// リスト項目のフォーマット
// モックアップ: シンプルな「時刻 名前」形式
- "ともだ おれ 様 / 202 · 1名 / [202]"
+ "15:00 田中太郎"
```
→ モックアップの方がシンプル。現在実装の情報量が多いので、どちらに寄せるか要判断。
  現在実装の方が実用的な情報（部屋番号・人数）を含んでいるため、モックアップ側を更新する選択肢もある。

## 備考

- **部屋名表示（#2）** はデータ構造の変更を伴う可能性あり。Supabaseの部屋テーブルに「display_name」的なフィールドがあるか確認が必要

- 比較はテストデータの違い（2024年7月 vs 2026年3月）があるため、日付固有の差異は無視している
